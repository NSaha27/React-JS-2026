##Q. Why do different Front-end frameworks exist?
Front-end frameworks exist for the following reasons -
To keep the UI in sync with the data coming from a server,
It provides us a correct way of structuring and writing our front-end code,
It helps developers and teams to be consistent over front-end design
##Q. What is React?
React is a Component-based, Declarative, Stateful library of JavaScript that is used to build complex User Interfaces (UI).

##Q. How to create a production ready React application?
Using Vite (a third-party build tool) we can create a production-ready React application. Use the following command to build a React application using Vite -

```bash
npm create vite@latest application_name
```

##Q. What does StrictMode of React library do?
It does the following two things -
It renders the root component twice in order to find certain bugs.
It checks if any outdated part of the React API is being used.

##Q. What are React Components?
React applications are entirely made out of React components.
React components are building blocks of any React application,
React components are pieces of UI that have their own data, logic and appearances,
Multiple components are combined to build complex User Interfaces,
React components are reusable, we can nest them to other components, and pass data into them via props.

##Q. What is JSX?
Declarative syntax to declare how a component should look like and how it works.
One component must return only one block of JSX code.
It is an extension of JavaScript code that embeds HTML, CSS, and JavaScript code and sometimes another React component to an HTML.
Each JSX code is converted to a React.createElement() function call.
We could use React without any JSX code.
